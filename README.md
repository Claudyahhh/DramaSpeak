# DramaSpeak

## 把英语口语，变成一场两个人共同推动的戏

DramaSpeak 是一个沉浸式双人英语剧场。两个人进入同一间线上房间，认领不同角色，在时间压力、信息差和隐藏任务中，用英语把剧情谈到结局。

它不限定只有职场题材：一局可以发生在董事会、发布会后台、暴雨庄园、深空科研站或婚礼前夜。但每个故事都必须满足同一个标准——冲突主要靠对话解决，并练到可以迁移到真实交流中的澄清、追问、说服、谈判、叙事、边界表达和情绪表达。

> 两个人，一间线上房间，一份只有自己看得到的任务卡。剧情会记住你们真正说过的话。

## 现有六部剧本

每部约 20–25 分钟。玩家拿到的是身份、立场、表达武器和隐藏目标，不是完整台词。

| 剧本 | 类型与剧情 | 主要英语能力 |
| --- | --- | --- |
| **《最后一班夜航》** | 峰会结束后的暴雨夜，危机公关与调查记者围绕一段录音交锋。 | 危机沟通、追问事实、争取时间、媒体应对 |
| **《董事会前夜》** | 收购案开会前，协议里突然多出一页无人承认的附录。 | 风险披露、合规表达、谈判让步、高层沟通 |
| **《消失的发布会 Demo》** | 发布会倒计时中核心 Demo 黑屏，创始人与投资人代表必须决定救项目还是揭开谎言。 | 项目危机、澄清、说服、对投资人表达 |
| **《遗嘱之夜》** | 暴雨庄园里，一份没读完的遗嘱把继承人与亡者助理留在同一间书房。 | 委婉质疑、动机澄清、关系边界、秘密谈判 |
| **《静默信号》** | 深空科研站只剩两名幸存者，氧气与事故日志都只允许一个答案。 | 证据核查、建立信任、危机决策、克制表达 |
| **《婚礼前夜》** | 婚礼前十小时，明天的新娘遇见三年前不辞而别的旧情人。 | 情绪表达、追问过去、设定边界、体面告别 |

## 一局怎么进行

**进房会合 → 选本选角 → 领取作战包 → 三幕开演 → 揭牌复盘**

1. 创建房间，把 6 位房间码发给搭档。
2. 房主选择剧本，双方分别认领角色。
3. 查看共同表达、角色专属表达和开口自测。
4. 三幕自由对话；系统同步台词、点亮表达，并可用 AI 生成幕间推进和 SOS 提示。
5. 结束后揭开隐藏任务，生成团队复盘，把有效表达存入个人复习本。

也可以用 AI 从真实经历、题材组合或一句话点子生成新的双人剧本。自创剧本只保存在创建者的浏览器中，进入房间后才通过受保护的房间数据分享给搭档。

## 在线体验

打开 [DramaSpeak](https://duo-stage-vercel.vercel.app)，把房间码发给可信任的搭档。推荐使用 Chrome，并通过微信语音或面对面交流；为了避免回声把对方声音重复转写，请佩戴耳机。

房间码是一把临时邀请钥匙，请不要公开发布。不要在对话或自创剧本中输入真实公司机密、客户信息、未公开项目或个人敏感信息。

## 本地开发

要求 Node.js 20 或更高版本。

```bash
npm install
npm run dev
```

常用验证命令：

```bash
npm test
npm run build
npm run check
```

没有配置远程存储时，静态页面和个人本地功能仍可打开，但创建双人房间会明确失败，不会静默降级为一个只在本机存在的“假房间”。

## 部署与环境变量

Vercel 构建配置位于 `vercel.json`。双人房间需要 Upstash Redis：

可复制 `.env.example` 作为本地配置起点；真实 `.env` 文件已被 Git 忽略。

| 变量 | 必需 | 说明 |
| --- | --- | --- |
| `UPSTASH_REDIS_REST_URL` | 是 | Upstash REST 地址 |
| `UPSTASH_REDIS_REST_TOKEN` | 是 | Upstash REST Token，只能保存在服务端 |
| `ROOM_TOKEN_SECRET` | 推荐 | 至少 32 字节的随机值，用于签发房间成员令牌；未设置时暂以 Redis Token 派生 |
| `ROOM_TTL_SECONDS` | 否 | 房间数据有效期，默认 86400 秒；代码限制为 1 小时至 7 天 |
| `STORAGE_NAMESPACE` | 否 | Redis 键命名空间；默认继续使用 `ai-drama` 以兼容已有数据 |
| `ALLOW_SERVER_MODEL` | 否 | 只有显式设为 `true` 才允许使用服务器模型 Key；默认关闭，避免公开额度被滥用 |

服务器模型为可选能力。启用 `ALLOW_SERVER_MODEL=true` 后，可按厂商配置 `TEXT_PROVIDER`、`TEXT_API_KEY`、`TEXT_MODEL`、`TEXT_BASE_URL`、`ANTHROPIC_API_KEY`、`GLM_API_KEY`，以及对应的 `IMAGE_*` / `GLM_IMAGE_*` 变量。`TEXT_MAX_TOKENS` 默认 1200，并被硬限制在 256–8000。未启用服务器模型且用户没有填写个人 Key 时，应用使用内置演示回退内容。

## 安全与数据边界

- 房间码只用于加入；加入后由服务端签发绑定“房间 + 用户 + 房主权限”的 HMAC 令牌。
- 普通成员只能修改自己的成员状态和台词；只有房主可以修改剧本、三幕状态和复盘。
- 存储接口不提供全局键扫描，只允许列出当前房间的最多两名成员。
- 房间 meta、成员、台词、动态剧情和复盘默认在最后活跃后 24 小时过期。
- 个人档案、备课进度、复习本和可选的模型 Key 保存在当前浏览器，不进入共享房间；自创剧本平时也只在本机保存，只有被房主选中后才进入受保护的房间数据供搭档读取。
- 文本 prompt 上限为 **100,000 字符**，约等于 25,000 个英文 token，足够容纳完整三幕对话，同时阻止多 MB 请求。
- 图片 prompt 上限为 **20,000 字符**。
- 文本接口每个来源 IP 最多 120 次/10 分钟；图片接口最多 12 次/10 分钟；创建或加入房间最多 30 次/10 分钟。
- 个人模型 Key 的请求会先经过 DramaSpeak 的 Vercel 服务端代理，再转发到对应模型服务商。Key 不会写入房间、复盘或 GitHub；勾选“记住”后会明文保存在该设备的 `localStorage`。
- 开启 AI 幕间、翻译或复盘时，相关剧本内容和必要的对话片段也会随请求发送给所选模型服务商。

## 代码结构

```text
api/                 Vercel serverless API：文本、图片、受保护的房间存储
lib/                 服务端安全、限流与共享常量
src/App.jsx          现役产品流程、剧本数据与界面
src/storage.js       浏览器本地存储和房间令牌客户端
tests/               Node 内置测试
vercel.json          构建与路由配置
```

项目当前仍是早期双人私用版本。生产数据模型或鉴权协议发生变化时，必须同步本 README、`AGENTS.md` 和安全测试。
